# 제품 개요

## 목적

학교 내 동아리(모임) 활동을 디지털로 기록하여 출석, 구성원 활동 데이터, 통계 등을 체계화하는 시스템을 제공한다.

## 사용자 종류

- 동아리 리더 (출석 관리자 또는 운영진)
- 동아리 일반 멤버

# 핵심 개념

- 모임(Club): 하나의 동아리 단위 (구름톤 유니브 한국외대)
- 세션(Session): 특정 일자에 출석을 받는 이벤트 단위
- 관리자(Admin): 세션 생성 및 출석 현황 확인/수정 권한 보유
- 이용자(Member): 출석 대상자, QR또는 핀코드로 출석 등록 가능
- 출석 현황: O(출석) / X(미출석) / △ (지각, 조퇴)
- n차 출석: 각 세션에 여러 번 출석하여 출튀를 방지 (관리자가 같은 세션에서 여러번 출석을 부를 수 있음)

# 핵심기능

## 1. 온보딩

- 로그인(IDP, Google, Naver SSO) 마지막으로 로그인 한 계정 띄우기
- 사용자 기본 정보 등록 (이름, 학번, 전공, 학교)

## 2. 주 기능

한 계정이 관리자 역할과 사용자 역할을 동시에 가질 수 있는 구조로, 메인 페이지에서 두 역할 기반으로 기능을 분리하는 UI로 구성

- 출석받기: 내가 관리자인 모임 리스트
    - 동아리 새로 만들기
- 출석하기: 내가 참여자인 모임 리스트
    - 동아리 참여하기 → 검색
    - 관리자에게 초대 링크를 요청하기

## 2.1. 관리자 기능(동아리 리더)

- 모임생성: 모임 이름 입력(필수), 그룹(부서)는 자동 부여: 부서 1, 부서 2 …
- 관리자 초대: 이메일로 초대 링크 발송 (30분 내 무효)
- 관리자 수정: 관리자 명단을 수정하고,  소유 권한을 넘길 수 있음
- 사용자 초대: 링크로 초대, 사용자가 수락 누르면 자동으로 등록
- 사용자 수정: 동아리 탈퇴, 휴면 처리
- 사용자 목록 조회: 동아리 연명부 형식으로 사용자를 table 형태로 표시
- 세션 생성: 현재 날짜, 및 시간으로 출석부 생성
- 출석 받기: QR코드 생성, 6자리 핀코드 제공
- 출석 수동 체크: UI 내에서 직접 체크 가능 (호명 출석용)
- 출석 현황 조회: 세션별 인원 출석 여부 조회 (토글)
- 출석 수정: 체크된 기록 변경
- 통계 보기: 사용자별 출석률, 세션별 출석률, 개근 TOP3, 지각 TOP3, 결석 TOP3

## 2.2. 이용자 기능

- 초대 수락: 초대 링크 클릭 후 → “수락” 누르면 모임에 자동 등록
- 출석하기: QR코드 스캔 또는 핀코드 입력
- 출석 상태 확인: 본인의 출석 내역 조회 기능

## 기타(Happy To)

- 보증금 기능: 세션 미출석 시 페널티
- 정기 활동 예정일을 미리 픽스해서 볼 수 있는 기능
- 출석 통계 CSV 다운로드
- 학기별/연간 통계 보고서
- 총학생회, 중앙 관리자 대시보드(동아리 위에 상위 개념을 만들어서 eg) 구름톤 중앙, 전체 스쿼드를 관리)
- 출석률 기반 뱃지, 포인트 시스템
